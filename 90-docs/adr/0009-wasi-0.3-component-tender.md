# ADR 0009: Kototama is the WASI 0.3 Component Model tender

Status: accepted; admission contract implemented, engine execution staged

## Decision

Kototama admits and composes compiler-produced WebAssembly components against
their exact WIT worlds. WASI 0.3.0 is the platform baseline. WASI 0.2.11 and
legacy core-Wasm execution remain explicit compatibility profiles and never
upgrade implicitly.

Kototama does not compile `.kotoba` and does not invent WIT. It verifies the
component target, upstream baseline, declared imports/exports, capability grant
intersection, provider bindings, and absence of undeclared or ambient WASI.

Sync WIT `func` remains valid under WASI 0.3. Async admission is separate:
`async func`, `future<T>`, and `stream<T>` require an async effect plus positive
cancellation, deadline, item, and byte budgets. Cancellation and budget
exhaustion converge on bounded runtime results; they cannot leave detached
provider work or an unbounded stream.

Provider components may receive WASI interfaces only through their declared
kit authority. Application components never receive filesystem, sockets,
clock, random, environment, or process authority from the target name alone.

The current Chicory/core-module tender remains legacy evidence. Component
qualification requires a Component Model engine with WASI 0.3 support and the
shared compiler/language semantic vectors.
