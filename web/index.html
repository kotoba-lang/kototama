kototama browser WASM AOT demo

All three elements below are defined by kotoba-lang/wasm-webcomponent (KotobaWasmElement + kgraph.js + actor-host.js) — this page no longer carries its own copy of that hosting code, it only supplies the `.wasm` payloads and a few short define() calls. This page itself is generated (see generate.cljs) via kotoba-lang/html/kotoba-lang/css (Hiccup-compatible EDN → HTML/CSS), run under nbb — the org's .cljc runtime priority (kototama > cljs > nbb > jvm) applied to authoring a static page, not just guest-side logic. See ADR-2607061630.

demo.wasm (zero imports)

Byte-for-byte output of kotoba wasm emit src/demo.kotoba in kotoba-lang/kotoba (73 bytes, exports main() : i32 which returns 42) — instantiated with the browser's native WebAssembly API, no JVM, no Chicory, no wasmtime.

demo-kgraph.wasm (kgraph-* host imports)

Byte-for-byte output of kotoba wasm emit src/demo_kgraph.kotoba --policy src/demo_kgraph_policy.edn in kotoba-lang/kotoba (219 bytes; declares the kgraph_assert/kgraph_query host imports), backed by kgraph.js's browser-side port of kotoba's in-memory EAVT datom store.

actor-host-demo.wasm (kototama.contract's actor:host ABI, partial)

Hand-assembled module importing clock_monotonic/log_write/sha256_hex (module "kotoba"), backed by actor-host.js — a browser-side port of kototama.contract's HostCaps/RuntimeLimits/validate-import-surface, the same fail-closed pre-flight + per-call grant checks kototama.tender (the JVM/Chicory counterpart) enforces. 7 of the 8 actor:host imports are implementable as SYNCHRONOUS Wasm host imports in a standard browser (gen-keypair/sign/verify are real too, via a vendored @noble/curves ed25519 — elliptic-curve signing is pure arithmetic, not I/O). Only http-post is missing: fetch is real network I/O, which a synchronous host import can't await — see actor-host.js's header comment for the honest scope note.
